## [1.2.0] — Visualizer Variants + Amplitude Boost

### Added
- **5 visualizer variants** via `variant` prop:
  - `line` — smooth bezier waveform (default, unchanged behavior)
  - `bars` — frequency bar chart using `getByteFrequencyData`
  - `mirror` — waveform reflected symmetrically top and bottom
  - `dots` — particle dot wave, dot size scales with amplitude
  - `circle` — radial waveform plotted around a circle
- **`VisualizerVariant`** constant exported from package root — use instead of raw strings for full TypeScript autocomplete
- **`VisualizerVariantType`** TypeScript type exported from package root
- **`fftSize` prop** — control `AnalyserNode.fftSize` (default `512`, was browser default `2048`)
- **`barSkipBins` prop** (`bars` only) — skip low-frequency DC offset bins that cause leftmost bars to jump in silence (default `4`)
- **`barSilenceThreshold` prop** (`bars` only) — minimum bin value `0–255` below which no bar is drawn, eliminates idle noise rendering (default `20`)
- **`reset()`** added to `useAudioRecorder` hook — resets state back to `NONE` cleanly
- **`fftSize`** and **`smoothingTimeConstant`** now passed through `AudioPulse` → `AudioAnalyserProvider` as props
- Variant GIF previews added to README (`waveform-line.gif`, `waveform-bars.gif`, `waveform-mirror.gif`, `waveform-dots.gif`, `waveform-circle.gif`)
- `src/renderers/` folder — each variant lives in its own isolated renderer file (`renderLine.ts`, `renderBars.ts`, `renderCircle.ts`, `renderMirror.ts`, `renderDots.ts`)

### Fixed
- **Bars variant idle jumping** — first 4 frequency bins (DC offset + hardware hum) are now skipped; bins below the silence threshold are not rendered
- **Wave amplitude too small** — `smoothingTimeConstant` default changed from `1` → `0.8`; `fftSize` default changed from browser default `2048` → `512`; all time-domain renderers now amplify deviation from centre by `2.5×`
- **`getByteFrequencyData` vs `getByteTimeDomainData`** — `bars` variant now correctly fetches frequency data; all other variants fetch time-domain data
- **`useRef` vs `createRef`** — `AudioVisualizer` was using `createRef` (new ref every render, breaking canvas). Changed to `useRef`

### Changed
- `smoothingTimeConstant` default: `1` → `0.8`
- `fftSize` default: browser default (`2048`) → `512`
- `AudioAnalyserProvider` now accepts `fftSize` prop in addition to `smoothingTimeConstant`

## [1.1.1] — TypeScript Rewrite + tsup

### Changed
- Entire package rewritten in **TypeScript** (`.tsx` / `.ts` throughout)
- Build tool changed from **Rollup** → **tsup** (esbuild-based, zero config, native TS support)
- `rollup.config.js` and all Rollup plugins removed
- `tsup.config.ts` added — replaces ~30 lines of Rollup config with ~15 lines
- `tsconfig.json` added
- `smoothingTimeConstant` default corrected to `0.8` (was hardcoded `1` in original JS)

### Added
- Full **TypeScript declarations** auto-generated by tsup (`dist/index.d.ts`) — no separate `@types/` package needed
- All types exported from package root: `AudioPulseProps`, `AudioResult`, `RecordStateType`, `UseAudioRecorderReturn`, context value types
- **`useAudioRecorder` hook** — `start`, `pause`, `stop`, `toggle`, `reset` — eliminates manual `RecordState` management
- **`fftSize` prop** on `AudioPulseProps` and `AudioAnalyserProvider`
- **`barSkipBins`** and **`barSilenceThreshold`** props
- `exports` field in `package.json` with correct condition order (`types` before `import`/`require`)
- `cjsInterop: true` + `splitting: true` in tsup config — fixes CJS named+default exports warning
- Type shim for `audio-recorder-polyfill` (`src/types/audio-recorder-polyfill.d.ts`)

### Fixed
- **`exports` condition order** — `types` must come before `import` and `require` in `package.json` exports map
- **`useRef<InstanceType<typeof AudioRecorder>>`** — was incorrectly typed as `typeof AudioRecorder` (the class constructor), causing TS errors on `.start()`, `.stop()`, `.pause()`, `.resume()`, `.addEventListener()`
- **CJS named + default exports warning** — resolved with `cjsInterop: true`

## [1.1.0] — Bug Fixes & Start Again

### Fixed
- **"Start Again" wave not showing** — `<AudioPulse>` was conditionally unmounted when audio was ready, tearing down all internal contexts. Fixed by keeping component always mounted and using `display: none` to hide it
- **`RecordState.NONE` unhandled** — state machine now resets `wave` to `false` on `NONE`, enabling clean restart
- **Wave frozen after resume** — `isPause` check in state machine was not reliably detecting pause state on remount

### Added
- `reset()` to `useAudioRecorder` — sets state back to `NONE`, required for clean "Start Again" flow
- `RecordState.NONE` case handled in `AudioVisualizer` state machine
- README updated with `display: none` pattern and `handleStartAgain` example using `reset()` + `setTimeout(start, 0)`

## [1.0.0] — Initial Release

### Added
- `AudioPulse` component — drop-in React audio recorder with real-time waveform canvas
- `RecordState` constant — `START`, `PAUSE`, `STOP`, `NONE`
- `useAudioRecorder` hook — `start`, `pause`, `stop`, `toggle`
- Waveform rendered on `<canvas>` using `quadraticCurveTo` bezier curves
- Idle flat line (30% opacity) shown when not recording
- Customizable `foregroundColor`, `backgroundColor`, `lineWidth`, `height`
- `renderVisualizer` render prop — full custom canvas control
- `smoothingTimeConstant` prop for analyser tuning
- MP3 output via `audio-recorder-polyfill` (cross-browser Safari + Firefox support)
- Internal contexts exported for advanced usage: `MediaStreamProvider`, `InputAudioProvider`, `AudioAnalyserProvider`
- Context hooks exported: `useMediaStream`, `useInputAudio`, `useAudioAnalyser`
- `AudioVisualizer` sub-component exported for custom layouts
- Full prop documentation in README
